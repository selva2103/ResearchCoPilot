STEP 1 — Expand/Collapse Transcript Details

Replace the flat transcript list (5.3A) with expandable transcript rows. 
Each transcript row, when clicked/expanded, reveals:
- Full accession + version
- Transcript type and status
- Length in nt + exon count
- MANE Select / MANE Plus Clinical badges (human only, per 5.3A rules — 
  do NOT show for non-human genes)
- Genomic location of this transcript (chromosome, start, end, strand) 
  — only if available from the already-fetched response, no extra API call
- "Download FASTA" button (now functional — see Step 2)
- "Download CDS" button for NM_/XM_ only (see Step 3)
- "Non-coding transcript" label for NR_/XR_ (see Step 3)

Collapsed state shows: accession prefix badge, accession + version, 
transcript type label, length in nt, status badge, MANE badge (if human).
Expanded state shows all of the above.

Accordion style by default — only one row expanded at a time. 
Do NOT auto-expand all transcripts on load.

--------------------------------------------------

STEP 2 — Transcript FASTA Download

Wire the "Download FASTA" button for each transcript. When clicked:
- Fetch transcript sequence via NCBI EFetch:
  db=nuccore, id={accessionVersion}, rettype=fasta, retmode=text
- Trigger browser download of the FASTA content
- Filename: {accessionVersion}.fasta (e.g. NM_000546.6.fasta)
- Fetch on demand only — do NOT pre-fetch on page load
- Show loading state on the button while fetching
- Show error state with retry option if fetch fails
- Every download call must go through the existing ncbi_entrez_limiter — 
  do not bypass the rate limiter for any reason
- If two FASTA downloads are triggered rapidly, the second must wait for 
  the limiter, not fire concurrently unthrottled

--------------------------------------------------

STEP 3 — CDS Retrieval and Non-coding Label

For NM_ and XM_ transcripts (protein-coding):
- Wire "Download CDS" button. When clicked:
  db=nuccore, id={accessionVersion}, rettype=fasta_cds_na, retmode=text
- Trigger browser download
- Filename: {accessionVersion}_CDS.fasta
- Same lazy-fetch, loading state, error state, and rate-limit rules 
  as Step 2

For NR_ and XR_ transcripts (non-coding RNA):
- Do NOT show a CDS button — not even disabled
- In the expanded detail panel, display the label "Non-coding transcript" 
  in a clearly visible position so users understand why CDS is absent
- This label is meaningful, not a placeholder — ncRNA transcripts 
  genuinely have no CDS

--------------------------------------------------

STEP 4 — Error Handling

Handle all of these states cleanly:
- Individual transcript expand failure (detail data unavailable) — show 
  error message inside the expanded row only, other rows unaffected
- FASTA download failure — error state on that button with retry, other 
  transcripts unaffected
- CDS download failure — same pattern as FASTA
- Rate limit hit during download — show "NCBI rate limit reached, please 
  wait a moment" near the affected button

Never crash the gene card or transcript list because one download failed.
Previously loaded transcript rows must always remain visible regardless 
of any individual operation's failure.
