WHAT THE PREVIOUS SESSION DISCOVERED (do not re-investigate these)

1. NCBI Gene EFetch with rettype=gene_table is lightweight (459 lines) 
   and contains transcript accessions, lengths, exon counts, and status 
   — but does NOT contain MANE Select information.

2. NCBI Gene EFetch with rettype=null (full XML, ~33MB for TP53) DOES 
   contain MANE Select. It is located at depth=4 in the XML under the 
   element Gene-commentary_label with value "MANE Select". The MANE 
   Select accession for TP53 is NM_000546.6.

3. The correct strategy is therefore: parse transcript list from 
   gene_table (lightweight), then extract MANE Select accession from 
   the full XML response. Both responses should be fetched once per gene 
   and reused — do NOT fetch them multiple times per request.

4. Mouse Trp53 has no MANE Select. isCanonical must be null (not false) 
   for all non-human transcripts. The MANE Select badge must not appear 
   for non-human genes.

5. Accession prefix → transcriptType mapping confirmed:
   NM_ → "mRNA", NR_ → "ncRNA", XM_ → "predicted_mRNA", 
   XR_ → "predicted_ncRNA", anything else → "other"
