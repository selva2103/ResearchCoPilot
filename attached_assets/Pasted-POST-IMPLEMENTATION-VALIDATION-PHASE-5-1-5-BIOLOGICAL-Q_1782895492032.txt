STEP 8 — BUG DISCOVERY AND IMMEDIATE FIX LOOP

This validation is NOT only for discovering bugs.

Its purpose is to leave the Biological Query Resolution Layer in a stable,
production-ready state.

If ANY bug is discovered during validation:

1. Stop and identify the exact root cause.

2. Fix the bug immediately using the smallest possible code change.
   Do NOT refactor unrelated code.
   Do NOT redesign the architecture.

3. Re-run ALL validation tests that could be affected by the fix,
   not only the failing query.

4. Confirm the original bug is resolved.

5. Verify that no regression has been introduced.

6. Continue the remaining validation from where it stopped.

If another bug is discovered later in the validation,
repeat this same Fix → Re-test → Continue cycle.

Do NOT postpone fixes until the end.

Do NOT leave known bugs unresolved.

Continue this iterative validation process until:

- all discovered bugs are fixed,
- all affected tests pass,
- and only genuine external limitations (for example NCBI rate limiting
  or missing upstream metadata) remain.

Only then produce the Final Report.
